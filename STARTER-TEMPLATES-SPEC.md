# Starter Templates Spec

เอกสารนี้สรุปแนวทางพัฒนา `Starter Templates` สำหรับแอปหมูนุ่น-เป๋าตุง โดยต่อยอดจากโค้ดปัจจุบันใน
- `app/starter-templates.tsx`
- `services/starterTemplateService.ts`

## เป้าหมาย

Starter Template ไม่ควรเป็นแค่ "เพิ่มหมวดหมู่เร็วขึ้น" แต่ควรเป็น `จุดตั้งต้นให้ระบบรู้จักผู้ใช้`

เมื่อผู้ใช้เลือก template แล้ว ระบบควรทำได้ 4 อย่างทันที
- สร้างหมวดรายรับ-รายจ่ายที่ตรงกับอาชีพนั้น
- Bias การเดา quick add ให้แม่นขึ้น
- แสดงตัวอย่างข้อความที่ผู้ใช้พิมพ์บ่อยในอาชีพนั้น
- เรียนรู้ต่อจากการแก้หมวดของผู้ใช้รายนั้น

## สภาพปัจจุบัน

ของเดิมใน `starterTemplateService.ts` มีข้อดีแล้ว 3 อย่าง
- มี template ใช้งานจริงได้
- กดใช้แล้วสร้างหมวดหมู่ได้ทันที
- กันหมวดซ้ำได้

แต่ยังมีข้อจำกัด
- โครงสร้าง template ยังแบนเกินไป มีแค่ `name`, `description`, `icon`, `categories`
- ยังไม่มี `group` สำหรับจัดหมวดหน้า UI
- ยังไม่มี `prediction bias` สำหรับช่วย parser
- ยังไม่มี `sample entries` สำหรับสอนผู้ใช้
- หน้า UI ยังแสดง template แบบยาวต่อเนื่อง ทำให้เริ่มแน่นเมื่อจำนวนอาชีพเพิ่ม

## ข้อเสนอหลัก

### 1. ใช้ Template 3 ชั้น

ระดับ A: `ยอดนิยม`
- แสดงบนจอแรก 6 รายการพอ

ระดับ B: `ดูทั้งหมดตามกลุ่มอาชีพ`
- บุคคลทั่วไป
- ค้าขาย/ธุรกิจ
- อาชีพอิสระ
- เกษตร/บริการ
- ครอบครัว/เฉพาะทาง

ระดับ C: `สร้างเอง`
- ใช้กับคนที่ไม่ตรง template ใด

### 2. Template หลักเวอร์ชันแรก

เริ่มจาก 12 template นี้ก่อน

1. นักศึกษา
2. พนักงานเงินเดือน
3. ครอบครัว
4. แม่ค้าออนไลน์
5. ขายของในตลาด
6. ธุรกิจส่วนตัว
7. ร้านอาหาร/คาเฟ่
8. ฟรีแลนซ์
9. Creator/ศิลปิน/นักดนตรี
10. Gamer/Streamer
11. เกษตรกร
12. ไรเดอร์/ขนส่ง

## รายละเอียด Template ที่แนะนำ

### บุคคลทั่วไป

#### นักศึกษา
- เหมาะสำหรับ: นักเรียน นักศึกษา และคนที่มีค่าใช้จ่ายการเรียนรายวัน
- รายรับหลัก: เงินสนับสนุนจากครอบครัว, ทุนการศึกษา, งานพิเศษ
- รายจ่ายหลัก: อาหารในมหาวิทยาลัย, เดินทางไปเรียน, หอพัก/ค่าเช่า, หนังสือ/อุปกรณ์เรียน
- คำที่ควรจับ: `ค่าขนม`, `พ่อโอน`, `แม่โอน`, `ค่าหอ`, `ค่าหนังสือ`

#### พนักงานเงินเดือน
- เหมาะสำหรับ: คนทำงานประจำทั่วไป
- รายรับหลัก: เงินเดือน, โบนัส, OT, ค่าคอมมิชชัน, รายได้เสริม
- รายจ่ายหลัก: อาหารกลางวัน, เดินทางทำงาน, ประกัน/สุขภาพ, กองทุน/ลดหย่อนภาษี
- คำที่ควรจับ: `เงินเดือน`, `โบนัส`, `ค่าเดินทางทำงาน`, `ประกัน`, `ภาษี`

#### ครอบครัว
- เหมาะสำหรับ: ผู้ดูแลค่าใช้จ่ายในบ้าน
- รายรับหลัก: เงินเดือนหลัก, เงินจากคู่สมรส, เงินช่วยเหลือครอบครัว
- รายจ่ายหลัก: ค่าอาหารในบ้าน, ค่าน้ำค่าไฟ, ของใช้ในบ้าน, ค่าลูก/การศึกษา, ค่าผ่อนบ้าน/รถ
- คำที่ควรจับ: `ค่ากับข้าว`, `ค่าลูก`, `ค่านม`, `ค่าเทอม`, `ของใช้ในบ้าน`

### ค้าขาย/ธุรกิจ

#### แม่ค้าออนไลน์
- เหมาะสำหรับ: ร้านค้าออนไลน์ ไลฟ์สด และขายผ่านโซเชียล
- รายรับหลัก: ยอดขายออนไลน์, ค่าส่งที่ลูกค้าจ่าย, รายได้ไลฟ์สด, affiliate
- รายจ่ายหลัก: ต้นทุนสินค้า, ค่ากล่อง/แพ็กของ, ค่าจัดส่ง, ค่าโฆษณา
- คำที่ควรจับ: `ออเดอร์เข้า`, `ลูกค้าโอน`, `ค่าส่งพัสดุ`, `ยิงแอด`, `ไลฟ์ขายของ`

#### ขายของในตลาด
- เหมาะสำหรับ: ร้านแผง ตลาดนัด ร้านเล็กหน้าร้าน
- รายรับหลัก: ขายผัก, ขายผลไม้, ขายของชำ, ขายอาหาร, ยอดขายหน้าร้าน
- รายจ่ายหลัก: ซื้อของเข้าร้าน, ค่าเช่าแผง, ค่าถุง/บรรจุภัณฑ์, ค่าขนส่งสินค้า, ค่าแรงลูกจ้าง
- คำที่ควรจับ: `ขายผัก`, `ขายผลไม้`, `ขายตลาด`, `ค่าแผง`, `ซื้อของเข้าร้าน`

#### ธุรกิจส่วนตัว
- เหมาะสำหรับ: เจ้าของกิจการขนาดเล็กถึงกลาง
- รายรับหลัก: รายได้จากธุรกิจ, ค่าบริการ, กำไรขายสินค้า
- รายจ่ายหลัก: ค่าวัตถุดิบ, เงินเดือนพนักงาน, ค่าเช่าร้าน, ค่าน้ำค่าไฟ, ภาษีธุรกิจ
- คำที่ควรจับ: `รายได้ธุรกิจ`, `เงินเข้าร้าน`, `จ่ายพนักงาน`, `ค่าเช่าร้าน`, `ภาษี`

#### ร้านอาหาร/คาเฟ่
- เหมาะสำหรับ: ร้านอาหาร เครื่องดื่ม และเดลิเวอรี
- รายรับหลัก: ขายอาหาร, ขายเครื่องดื่ม, เดลิเวอรี, รับจัดเลี้ยง
- รายจ่ายหลัก: วัตถุดิบ, น้ำแข็ง/แก๊ส, บรรจุภัณฑ์, ค่าแอปเดลิเวอรี, ค่าแรงพนักงาน
- คำที่ควรจับ: `ขายกาแฟ`, `ขายอาหาร`, `ค่าวัตถุดิบ`, `ค่าแก๊ส`, `เดลิเวอรี`

### อาชีพอิสระ

#### ฟรีแลนซ์
- เหมาะสำหรับ: รับงานโปรเจกต์และงานจ้างอิสระ
- รายรับหลัก: รายได้โปรเจกต์, มัดจำงาน, รับงานออกแบบ, รับตัดต่อ, รับเขียนโปรแกรม
- รายจ่ายหลัก: ค่าเครื่องมือทำงาน, ค่า software, ค่าเดินทางพบลูกค้า, ภาษีหัก ณ ที่จ่าย
- คำที่ควรจับ: `รับงาน`, `มัดจำงาน`, `ค่าตัดต่อ`, `ลูกค้าจ่าย`, `หัก ณ ที่จ่าย`

#### Creator/ศิลปิน/นักดนตรี
- เหมาะสำหรับ: นักดนตรี นักร้อง ศิลปิน และ content creator
- รายรับหลัก: ค่างานแสดง, เล่นดนตรี, รายได้จาก content, sponsor, โดเนท
- รายจ่ายหลัก: ค่าอุปกรณ์, ค่าแต่งตัว, ค่าเดินทาง, ค่าตัดต่อ, ค่าโปรโมต
- คำที่ควรจับ: `รับงานเล่นดนตรี`, `ร้องเพลงงาน`, `โดเนท`, `สปอนเซอร์`, `ค่าทีมงาน`

#### Gamer/Streamer
- เหมาะสำหรับ: คนทำช่องเกม ไลฟ์สตรีม และรายได้จากแพลตฟอร์ม
- รายรับหลัก: สตรีมเกม, โดเนท, sponsor, affiliate, รายได้จากแพลตฟอร์ม
- รายจ่ายหลัก: อุปกรณ์คอม, เกม/subscription, อินเทอร์เน็ต, ค่ากราฟิก, ค่าโฆษณาช่อง
- คำที่ควรจับ: `สตรีมเกม`, `โดเนท`, `สปอน`, `ซื้อเกม`, `อินเทอร์เน็ต`

### เกษตร/บริการ

#### เกษตรกร
- เหมาะสำหรับ: ผู้ใช้ที่มีรายรับจากผลผลิตและงานเกษตร
- รายรับหลัก: ขายผัก, ขายผลไม้, ขายข้าว, ขายสัตว์เลี้ยง/ปศุสัตว์
- รายจ่ายหลัก: เมล็ดพันธุ์, ปุ๋ย/ยา, ค่าแรง, ค่าน้ำมันเครื่องจักร, ค่าขนส่งผลผลิต
- คำที่ควรจับ: `ขายผัก`, `ขายข้าว`, `ขายไข่`, `ซื้อปุ๋ย`, `ค่าน้ำมันรถไถ`

#### ไรเดอร์/ขนส่ง
- เหมาะสำหรับ: ส่งอาหาร ส่งพัสดุ และขับรถรับจ้าง
- รายรับหลัก: วิ่งงาน, ค่าส่งอาหาร, ค่าส่งพัสดุ, ค่าโดยสาร
- รายจ่ายหลัก: ค่าน้ำมัน, ค่าซ่อมรถ, ค่าโทรศัพท์, ค่าอินเทอร์เน็ต, ค่าผ่อนรถ
- คำที่ควรจับ: `วิ่งงาน`, `ค่าส่งอาหาร`, `ค่าโดยสาร`, `เติมน้ำมัน`, `ซ่อมรถ`

## โครงสร้างข้อมูลที่ควรขยาย

จากเดิม

```ts
type StarterTemplate = {
  id: string;
  name: string;
  description: string;
  icon: string;
  categories: StarterTemplateCategory[];
};
```

ควรขยายเป็น

```ts
type StarterTemplateV2 = {
  id: string;
  name: string;
  shortDescription: string;
  icon: string;
  group: 'popular' | 'general' | 'commerce' | 'freelance' | 'agriculture' | 'family';
  targetUser: string;
  popularity: number;
  categories: StarterTemplateCategory[];
  suggestedWallets?: Array<{
    name: string;
    icon?: string;
    type?: 'cash' | 'bank' | 'ewallet';
  }>;
  sampleEntries: string[];
  predictionBias: {
    preferredIncomeCategories: string[];
    preferredExpenseCategories: string[];
    keywordMappings: Record<string, string>;
  };
  helperTags: string[];
};
```

### ฟิลด์ใหม่ที่สำคัญ

`group`
- ใช้จัดหมวด UI และ filter ด้านบน

`targetUser`
- ใช้แสดงข้อความ `เหมาะสำหรับใคร`

`sampleEntries`
- ใช้สอนผู้ใช้และทำ onboarding

`predictionBias`
- ใช้ช่วย quick add parser เช่น ถ้าเลือก `เกษตรกร` แล้วพิมพ์ `ขายผัก` ให้ bias ไป `เกษตรกร` หรือ `ขายของในตลาด` ก่อนหมวดกว้าง

`helperTags`
- แสดง chip สั้น 4-5 รายการบนการ์ดแทนการเอาหมวดทั้งหมดขึ้นครบ

## ผลกระทบต่อระบบเดาอัตโนมัติ

Starter Template ควรผูกกับ parser 4 ด้าน

1. Bias ประเภท
- ถ้า template หลักเป็น `แม่ค้าออนไลน์` ให้ parser เชื่อโลก `income` จากคำ `ลูกค้าโอน`, `ออเดอร์เข้า`, `ยอดขาย`

2. Bias หมวดหมู่
- ถ้า template หลักเป็น `เกษตรกร` ให้คำ `ขายผัก`, `ขายไข่`, `ขายข้าว` ชนะ `ฟรีแลนซ์`

3. ตัวอย่างข้อความ
- ใช้เป็น placeholder/help ในหน้าเพิ่มรายการ เช่น `ขายเสื้อ 590`, `รับงานตัดต่อ 2500`, `ซื้อปุ๋ย 780`

4. User Learning
- เมื่อผู้ใช้แก้หมวดจาก template ใด ให้บันทึกพฤติกรรมนั้นเป็น personal rule ต่อ template นั้นได้

## UI ที่แนะนำ

หน้า `starter-templates.tsx` ควรปรับจาก list ยาวแบบเดียว เป็น layout นี้

### ส่วนบน
- Header: `Starter Templates`
- Search bar: `ค้นหาอาชีพหรือกิจกรรม...`
- Filter chips:
  - ยอดนิยม
  - พนักงาน
  - ค้าขาย
  - ฟรีแลนซ์
  - เกษตร
  - ครอบครัว
  - อื่น ๆ

### ส่วนกลาง
- Section `ยอดนิยม`
- แสดง 6 template แรก
- Section `หมวดอาชีพทั้งหมด`
- แสดงเป็นการ์ดตามกลุ่ม

### ส่วนล่าง
- ปุ่ม `สร้างเอง`
- ปุ่ม `ให้ระบบช่วยเลือก`

## การ์ด Template ที่สะอาดขึ้น

การ์ดแต่ละใบควรมีแค่ 5 ส่วน
- ไอคอน
- ชื่ออาชีพ
- คำอธิบายสั้น 1 บรรทัด
- helper tags 4-5 อัน
- ปุ่ม `ใช้ Template นี้`

### ตัวอย่างข้อความบนการ์ด

`พนักงานเงินเดือน`
- เงินเดือน โบนัส ค่าเดินทาง และค่าใช้จ่ายประจำ

`เกษตรกร`
- ผลผลิตตามฤดูกาล ปุ๋ย ค่าแรง และค่าน้ำมันเครื่องจักร

### ข้อควรลด
- อย่าแสดงหมวดทุกอันครบใน card
- ถ้าเกิน 5 chip ให้แสดง `+ อีก 2 หมวด`
- ลด description ให้เหลือ 1 บรรทัด

## ฟีเจอร์ที่ควรเพิ่มถัดไป

### 1. Template หลัก + โปรไฟล์เสริม

ผู้ใช้ไทยจำนวนมากมีมากกว่า 1 บทบาท เช่น
- หลัก: พนักงานเงินเดือน
- เสริม: แม่ค้าออนไลน์

ระบบควรรองรับ
- `primaryTemplateId`
- `secondaryTemplateIds[]`

### 2. ให้ระบบช่วยเลือก

เพิ่มการ์ดพิเศษ `ให้ระบบช่วยเลือก`

ถาม 3 ข้อ
- รายได้หลักของคุณมาจากอะไร
- คุณขายของ / รับจ้าง / เงินเดือน / เกษตร / อื่น ๆ
- รายจ่ายที่เกิดบ่อยคืออะไร

แล้วแนะนำ template ให้เลย

### 3. แนะนำ template ซ้อนจากพฤติกรรม

ถ้าผู้ใช้เลือก `พนักงานเงินเดือน` แต่พิมพ์คำแนว `ออเดอร์`, `ลูกค้าโอน`, `ยิงแอด` บ่อย
- แนะนำ `เพิ่มโปรไฟล์เสริม: แม่ค้าออนไลน์`

## แผนพัฒนาแนะนำ

### Phase 1
- เพิ่ม 12 template หลัก
- เพิ่ม `group`, `helperTags`, `sampleEntries`
- ปรับ UI เป็น list + filter chip

### Phase 2
- เพิ่ม `predictionBias` ต่อ template
- ผูก bias เข้ากับ quick add parser

### Phase 3
- เพิ่ม search
- เพิ่ม `ให้ระบบช่วยเลือก`
- เพิ่ม `template หลัก + โปรไฟล์เสริม`

## สรุปการตัดสินใจ

ถ้าจะเริ่มทำตอนนี้ ให้โฟกัส 4 เรื่องก่อน
- เพิ่ม 12 template ให้ครอบคลุมตลาดไทย
- ขยาย data model ของ template ให้ช่วย parser ได้
- ปรับหน้า UI ให้มี `ยอดนิยม + filter + search`
- รองรับการเรียนรู้ต่อจาก template และการแก้หมวดของผู้ใช้
