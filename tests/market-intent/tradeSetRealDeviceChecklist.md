# Real-Device Checklist — Trade Set Save + Report QA

ใช้หลัง `npm run test:market-intent` ผ่านแล้ว เพื่อตรวจสิ่งที่ unit test เห็นไม่ครบ: การบันทึกจริง, ประวัติ, รายงาน, และ learning หลังผู้ใช้แก้หมวด

## A. ก่อนกดบันทึก: การ์ด Trade Set

- [ ] `รับซื้อหน่อไม้จากสวน 4,000 บาทไปวางขายในตลาดได้ 9,700 บาท`
  - การ์ด: `ชุดซื้อ-ขาย`
  - ความมั่นใจ: `มั่นใจสูง`
  - ต้นทุน: `-4,000`
  - ยอดขาย: `+9,700`
  - กำไรสุทธิ: `+5,700`
  - หมวดที่เห็น: `ขายของในตลาด`

- [ ] `รับผักสดจากชุมชนจ่ายไป 1,300 บาทไปขายในตลาดได้ 3,400 บาท`
  - ต้นทุน: `-1,300`
  - ยอดขาย: `+3,400`
  - กำไรสุทธิ: `+2,100`
  - หมวดที่เห็น: `ขายของในตลาด`

- [ ] `ซื้อชุดทำส้มตำ 800 บาทเร่ขายรถเข็นในชุมชนได้ 2,500 บาท`
  - การ์ด: `ชุดซื้อ-ขาย`
  - ความมั่นใจ: `มั่นใจปานกลาง`
  - หมวดที่เห็น: `ธุรกิจส่วนตัว`
  - เหตุผล: ไม่ได้พูดตลาดชัด จึงไม่ควรถูกดันเป็น `ขายของในตลาด`

## B. หลังบันทึกจริง: ประวัติ

สำหรับแต่ละเคสที่กด `บันทึกชุดซื้อ-ขาย · 2 รายการ`:

- [ ] มีธุรกรรม 2 รายการ ไม่ใช่รายการเดียว
- [ ] รายจ่ายมี `tradeRole=cost` และหมวด `ต้นทุนขาย`
- [ ] รายรับมี `tradeRole=revenue` และหมวดตามการ์ด เช่น `ขายของในตลาด` หรือ `ธุรกิจส่วนตัว`
- [ ] ทั้งสองรายการผูก `tradeGroupId` เดียวกัน
- [ ] note ของรายจ่ายลงท้าย `(ต้นทุน)` และรายรับลงท้าย `(ยอดขาย)`

## C. รายงานและกำไร

- [ ] กำไรไม่ถูกบันทึกเป็น transaction แยก
- [ ] รายงานรายรับนับเฉพาะยอดขาย gross
- [ ] รายงานรายจ่ายนับเฉพาะต้นทุนขาย
- [ ] กำไร derive จากรายรับ - ต้นทุน ไม่ถูกนับซ้ำ
- [ ] เคสขาดทุน เช่น `ซื้อเสื้อมา 2,000 ขายได้ 1,500` แสดงกำไรเป็น `-500` หรือขาดทุนอย่างถูกต้อง

## D. Learning หลังแก้หมวด

- [ ] ใช้เคสที่ระบบเสนอ `ธุรกิจส่วนตัว`
- [ ] แก้หมวดบนการ์ดเป็น `ขายของในตลาด`
- [ ] กดบันทึก
- [ ] พิมพ์ประโยคเดิมหรือใกล้เคียงมากอีกครั้ง
- [ ] ระบบเสนอหมวดที่แก้ไว้ดีขึ้น แต่ยังไม่ควร override amount/type ที่ core parser จับได้

## E. สิ่งที่ควรจดเป็น real log

ถ้าพบเคสผิด ให้จด 5 อย่างนี้:

- ข้อความที่พิมพ์จริง
- หมวดที่ระบบเสนอ
- หมวดที่ควรเป็น
- ยอดต้นทุน/ยอดขายที่ระบบจับได้
- หลังบันทึก รายการในประวัติถูกแยก 2 ขาหรือไม่

นำ real log กลับมาเพิ่มใน `tests/market-intent/cases.market-intent.json` ก่อนเพิ่ม rule ใหม่ทุกครั้ง
